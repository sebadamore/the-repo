Stormboard Whiteboard

Data-first digital whiteboard. Conservative sharing posture: no public URLs, email-only invites, strict Member / Guest license separation.

Permissions

Member/Guest × Administrator/Contributor/Viewer/Guest Contributor. Two-axis model splits billing from permission.

Public link

None. All external access requires email-invited Guest account.

Guest collaboration

Email invite. Contribute requires Enterprise + Guest Pass (one Storm at a time).

For the Guest Pass banner, per-invitee role picker, and member-upgrade flow — live help article shows them freshly.

Miro Whiteboard

Category leader. Mature, Google-Docs-style sharing. 3 roles, anonymous public links, Visitors that don't need an account.

Permissions

3 roles: Editor / Commenter / Viewer. Default sharing policy set per team/company.

Public link

"Anyone with the link" → Can view / comment / edit. Password optional. Free caps at view.

Guest collaboration

Visitors follow public link — no account. Enterprise admins can disable org-wide.

For the actual "Anyone with the link" toggle with the password affordance and Visitor join flow, see Miro's live help article.

FigJam (Figma) Whiteboard

Inherits Figma's battle-tested sharing dialog. Public links with password, admin-level kill-switch, open-session controls.

Permissions

Can view / Can edit at file, project, team, and org levels — with inheritance + explicit overrides.

Public link

"Anyone with the link" or "Anyone with link + password." Org admins can disable entirely.

Guest collaboration

Open sessions let non-signed-in users edit. Viewers are free; Editors consume a seat.

For the actual share-dialog UI (role dropdown, password modal, link-sharing toggle), see the live help article.
Open live help article →

Lucidspark Whiteboard

Lucid's whiteboard. Notable for its "Guest Collaborator link" — unique in letting non-account holders edit on Enterprise.

Permissions

4 levels: View / Comment / Edit / Edit and Share. More granular than Miro.

Public link

Shareable link with role. "Anyone with the link" surfaces vary by plan/chart type.

Guest collaboration

Enterprise Guest Collaborator link: fully edit without a Lucid account or paid seat. Unique in this set.

Whimsical Whiteboard

Light, IA-focused collaboration tool. Anonymous comments/edits via toggle, hard guest caps as a pricing lever.

Permissions

3 roles + "No access." Inheritance from folder to file; per-file override.

Public link

Airplane icon → view/comment/edit. Password protection. "Include board position" pin.

Guest collaboration

Hard per-file caps: Free 10, Pro 50, Business 100, Enterprise 200. Anonymous edits via toggle.

tldraw Whiteboard

Lowest-friction whiteboard in the set. No login, no accounts — drawings are shared by URL, room-based real-time collaboration.

Permissions

Near-zero in hosted app — URL holders co-edit. SDK exposes editor/viewer roles via auth layer.

Public link

The URL is the permission. Unique room IDs, up to ~50 concurrent users.

Guest collaboration

Everyone is a guest. No seat accounting, no admin controls.

tldraw deliberately ships no share dialog on the hosted app — the URL is the permission. These GIFs from the tldraw sync launch are the closest to a "sharing UX" shot.

Notion Docs

Docs + database hybrid. Most granular permission model in the set — 4 tiers, page-level inheritance, and link expiry built-in.

Permissions

4 levels: Full access / Can edit / Can comment / Can view. Inheritance from parent pages; broadest access wins.

Public link

"Anyone on the web with link" — view/comment/edit. Optional expiry date. Publish-to-web is separate.

Guest collaboration

Email-invited guests scoped to pages. Don't consume Member seats. "Full access" guest can remove workspace owners — footgun.

Evernote Notes

Legacy note app. Note-level public links work; notebook-level public links do not — meaningful asymmetry for sharing workflows.

Permissions

Notes: view/edit/edit+invite. Notebooks: same roles but recipients must have Evernote accounts.

Public link

Per-note only. "Anyone with link can view" and "can edit" — edit happens in reduced Lite editor. Notebooks: no public URL.

Guest collaboration

No free-guest tier. External notebook collaborators are full invitees.

Canva Design

Design surface with the broadest consumer footprint. Sharing feels like Google Docs but layered with team-discovery and brand controls.

Permissions

Can edit / Can comment / Can view. Scopes: only-you / only-team / anyone-with-link. Team-discovery toggles search visibility.

Public link

Anyone with the link — edit, comment, or view. Anonymous edit works with no Canva account.

Guest collaboration

No seat cost for link-followers. Restrict Design Sharing controls prevent re-sharing externally (Teams/Enterprise).

Lovable Builder

AI app builder. Clean split between "publish the app" (public URL, no code access) and "share the project" (editor access).

Permissions

4 roles: Viewer / Editor / Admin / Owner. Clear escalation — Owners transfer/delete, Admins manage settings, Editors build.

Public link

Published URL ≠ editor access. App is public at URL, source/editor gated. Enterprise: restrict external publishing to owners/admins.

Guest collaboration

Email invite only. No free-guest tier. Project-level or workspace-level scopes. Credits charged to owner's workspace.

Lovable's own docs use text-only permission tables. These launch-post shots are the closest to a real "share dialog" — for the latest UI, see the live changelog and collaboration docs.

Perplexity AI Research

AI answer engine. Two distinct primitives — Thread (single Q&A, link-shareable) and Space (invited collaborators with roles).

Permissions

Threads: private/sharable. Spaces: Viewer / Contributor. Enterprise adds org admin / member / read-only.

Public link

Thread "Sharable" toggle — anyone with link can view anonymously. No edit for Threads; view only.

Guest collaboration

Space invites by email or link. Contributor can create threads, Viewer read-only. No seat economics.

Perplexity's help center is JavaScript-rendered. The live article shows the Share button location, Sharable toggle, and role picker for Space invites.

Claude AI Assistant

Projects bound to the organization. "Public" means org-visible — never open internet. Individual chats shareable via private links.

Permissions

2 levels: Can use / Can edit. No viewer-only tier. Project visibility: Private or Public-in-org.

Public link

"Public" ≠ internet-public — means every org member can see it. Individual chat shares via private link.

Guest collaboration

No external-guest model. All collaborators are org members.

For the Share button / General-access dropdown / bulk-email invite modals (the actual sharing toggles), see the live help article.

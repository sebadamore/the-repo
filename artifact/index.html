Patterns that emerge when you read all 12 side-by-side.

Permissions model

3–4 roles is the de-facto standard

Miro, Notion, Canva, Whimsical, Figma all converge on Viewer / Commenter / Editor. Notion adds "Full access"; Lucidspark adds "Edit + share"; Miro keeps it strictly 3.

Billing-vs-access is a separate axis

Stormboard and Lucid separate "Member (paid seat)" from "Guest (free)" from the Storm-level permission. Miro's "Visitor" concept is similar. Notion, Canva, Figma collapse both axes into one role picker.

Dangerous defaults exist

Notion: a Guest granted Full access can remove the Workspace Owner from a page. Miro/Figma default to "Can edit" when creating public links on paid plans.

Real-time tools lean open

tldraw and Lovable start from "share the URL, you're in." Miro, FigJam, Whimsical follow the Google-Docs mental model. Stormboard is the outlier that still requires an invited account.

Public / anonymous link

"Anyone with the link" is table stakes — except Stormboard & Evernote notebooks

10 of 12 offer anonymous view via URL; most also offer anonymous edit. Stormboard requires email-based Guest invite; Evernote has no public URL for notebooks (notes yes).

Password + expiry are the enterprise signal

Figma, Notion, Whimsical ship both. Miro ships password-only. Canva/Lovable lean on published-vs-shared as a separate affordance.

Admin kill-switch is now standard

Figma and Miro let organization admins disable public links entirely. Stormboard's enterprise posture is stricter by default — no public sharing to disable in the first place.

"Published" ≠ "shared" in builders

Lovable and Canva distinguish the public-facing output (a website / design) from editor access to the source. This is an honest model newer to the whiteboard category.

External / guest collaboration

Free-guest economics diverge widely

Lucidspark and Stormboard (Enterprise) let unlimited Guests contribute without a seat. Whimsical caps at 10 (Free) / 50 (Pro) / 100 (Business) / 200 (Enterprise) guests per file. Miro's "Visitor" via public link is effectively unlimited but read/comment-only on Free.

"No account needed" is the UX signal

tldraw, Miro Visitors, Whimsical anonymous-edit, Canva public link, Evernote note link — all let someone work without signing up. Stormboard, Claude, Notion require an account.

Per-guest roles are rarer than you'd think

Lucid and Stormboard Enterprise offer per-guest role selection. Most others treat all link-followers identically, with the link itself carrying the permission.

Collaboration surfaces vary by primitive

Perplexity: Thread (public link) vs Space (invited roles). Claude: Project (org-visible) vs Chat (private link). Notion: page or entire workspace. The whiteboard tools standardize on the Board as the unit.
